How can we install and use Lapack in visual studio?

You definitely can, I have done it before.

Visual Studio is a proprietary software that cost between 500$ to 5000$ if you have access to it it will be good but there are things that need to know about using VS (Visual Studio)

As long as I know VS does not support gfortran compiler, this means that you have have to purchase a fortran compiler from a vendor that is supported by VS.

VS supports intel fortran or ifort. Intel also provides its own version of lapack and called mkl-lapack.

In summary in order to be able to call lapack subroutines and work with VS you should have access to ifort and mkl-lapack on your system (it is not going to be cheap) and you have to link your source code with mkl-lapack.

The linking process inside VS is a little complicated you have to follow the linking advisor of mkl-lapack and set it based on the version of your fortran and mkl and also your machines architecture. (https://software.intel.com/en-us/articles/intel-mkl-link-line-advisor)

After finding the correct linking you have to add it to your linking chain in VS. I have done it once. There was a sub menu in one of the several sub menus of VS (I had problem solving the conflicts between mkl and other packages needed to compile my code in VS I could solve it somehow).

You can check this website for more information but I am almost sure that you have to have different setting based on the your machine and also the version of your compiler: http://icl.cs.utk.edu/lapack-for-windows/lapack/

In general even thought it will be possible to run the code on VS i do not recommend it since the time that you might spend in setting up the VS for lapack will be more that you expect.

The other thing about combination VS, ifort and mkl-lapack is that it is not going to be free and it will cost lots of money if you want to install it on your laptop for example.
